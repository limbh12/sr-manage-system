# 프로젝트 변경 이력 (History)

이 문서는 프로젝트의 주요 변경 사항을 기능별로 기록합니다.

## 2025-12-06

### 1. 다크모드 기능 구현 (Dark Mode Feature)
- **Redux 상태 관리**:
  - `themeSlice.ts` 생성: 라이트/다크 모드 상태 관리를 위한 Redux slice 구현.
  - 로컬스토리지 연동: 사용자의 테마 설정을 `localStorage`에 저장하여 재방문 시에도 유지.
  - 시스템 설정 감지: 처음 방문하는 사용자는 브라우저의 `prefers-color-scheme` 설정을 자동으로 감지하여 초기 테마 적용.
  - `store/index.ts` 업데이트: `themeReducer`를 Redux 스토어에 등록.

- **CSS 테마 시스템**:
  - CSS 변수 기반 테마: `App.css`에 라이트 모드와 다크 모드용 색상 변수 정의.
  - 라이트 모드: 흰색 배경(`#ffffff`), 밝은 텍스트(`#333333`), 밝은 테두리(`#e0e0e0`).
  - 다크 모드: 어두운 배경(`#1e1e1e`), 밝은 텍스트(`#e0e0e0`), 어두운 테두리(`#333333`).
  - 부드러운 전환: `transition` 속성을 활용하여 테마 전환 시 자연스러운 애니메이션 효과 적용.
  - 전역 스타일 업데이트: 모든 주요 UI 요소(카드, 테이블, 폼, 모달, 버튼 등)에 CSS 변수 적용하여 테마 변경 시 자동 반영.

- **테마 토글 컴포넌트**:
  - `ThemeToggle.tsx` 생성: 다크모드/라이트모드 전환 버튼 컴포넌트 구현.
  - 아이콘 UI: 라이트 모드에서는 달 아이콘, 다크 모드에서는 태양 아이콘을 표시하여 직관적인 UX 제공.
  - 접근성: `aria-label`과 `title` 속성을 추가하여 스크린 리더 사용자와 키보드 네비게이션 지원.
  - `ThemeToggle.css`: 호버 및 클릭 효과를 포함한 버튼 스타일 정의.

- **헤더 통합**:
  - `Header.tsx` 수정: 헤더의 우측 상단에 `ThemeToggle` 컴포넌트 추가.
  - 사용자 정보 및 로그아웃 버튼과 함께 배치하여 쉽게 접근 가능.

- **앱 레벨 테마 적용**:
  - `App.tsx` 수정: Redux 스토어에서 테마 상태를 구독하고, `useEffect`를 통해 HTML 루트 요소에 `data-theme` 속성 설정.
  - 테마 변경 시 전체 애플리케이션에 즉시 반영되도록 구현.

### 2. 목적 및 효과
- **사용자 경험 향상**: 사용자가 선호하는 테마를 선택하여 눈의 피로를 줄이고 가독성 향상.
- **접근성 개선**: 시스템 설정을 자동으로 감지하여 별도 설정 없이 사용자의 환경에 맞춤.
- **일관성**: 모든 페이지와 컴포넌트에서 통일된 테마 적용으로 일관된 UI/UX 제공.

- **스타일**:
  - `frontend/src/App.css` (수정 - CSS 변수 및 다크모드 스타일 추가)
- **앱 진입점**:
  - `frontend/src/App.tsx` (수정 - 테마 적용 로직 추가)

### 4. 사용 방법
- 헤더 오른쪽 상단의 달/태양 아이콘 버튼을 클릭하여 다크모드와 라이트모드 전환.
- 테마 설정은 로컬스토리지에 자동 저장되어 다음 방문 시에도 유지.
- 처음 방문 시 브라우저의 시스템 설정에 따라 자동으로 테마 선택.
